Sudoku Solver API
by Samuel Charles Bird

# About

This is a project that has helped me learn about creating a simple API backend with Python and FastApi whilst improving my writing of algorithms.

In 2021 I built a purely front end sudoku solver (take a look for yourself) using a very modest stack: HTML, Sass and Typescript. There were some problems with this project: sometimes it couldn't solve the sudoku; the code wasn't well organised; I was dealing with UI functionality in the same code that was solving the puzzle.

Fast-forward to 2022 and I'm taking an interest in back end web development and this API is born. I have rethought and reorganised the code and success (!), there appears to be no bugs and the algorithm works as intended. Additionally whilst debugging and testing the code I noticed a pattern which lead me to a new algorithm that is much more efficient.

# Endpoints

/solve

To use this API to solve a sudoku you must sumbit a HTTP POST request to this endpoint and the request body must contain a valid input. The following JSON object is a valid input: 

{
  "puzzle": [
    0, 7, 0, 0, 0, 0, 0, 0, 0, 5, 3, 0, 0, 0, 0, 0, 0, 0, 0, 0, 2, 0, 0, 4, 1, 0, 0, 0, 0, 0, 2, 0, 0, 7, 3, 0, 8, 0, 0, 0, 0, 0, 0, 2, 6, 0, 0, 1, 0, 0, 5, 4, 0, 0, 3, 0, 8, 0, 0, 0, 0, 0, 0, 6, 0, 0, 1, 0, 7, 0, 0, 3, 0, 0, 0, 0, 0, 0, 0, 8, 0
  ]
}

/solve/?v1=true

By default, when just using /solve, this endpoint will use Version 2.0 of my solving algorithm. This is the fastest and most efficient option. However, you can use Version 1 by specifying a query in the URL as demonstrated just above.
